# P5.6 — empirical σ_MA → Yukawa-eigenvalue propagation

**Closes**: brutal-review §4.2 concern that "if σ→Yukawa propagation is roughly linear, then σ ≲ 10⁻³ implies Yukawa bias ≲ 10⁻³, sufficient for 5σ on PDG observables" was untested hand-waving.

**Status**: empirical study complete. Verdict differs from the brutal-review hypothesis.

---

## Geometry pivot

The task brief asked for the Fermat quintic at k=2. **This codebase has no Yukawa pipeline on the Fermat quintic** — the σ-MA evaluator (`QuinticSolver` in `src/quintic.rs`) and the Yukawa pipelines (`yukawa_overlap::compute_yukawa_spectrum` and `route34::yukawa_pipeline::predict_fermion_masses`) live on different geometries. The Yukawa pipelines operate on the **bicubic Tian-Yau CY3** (CP³ × CP³).

Building a Fermat-quintic Yukawa pipeline from scratch is well outside the P5.6 scope (it would require zero-mode bundles on CP⁴, polynomial seeds, harmonic projection, triple-overlap quadrature, and an SU(5)-or-similar GUT decomposition — months of work). So we measured on Tian-Yau where **both σ_MA and Yukawa are computable**.

The σ ladder is generated by varying the Donaldson optimisation budget (`max_iter`) at fixed seed and sample cloud, with `donaldson_tol = 1e-9` to prevent early termination. The Yukawa eigenvalues we track are the **9 fermion masses at M_Z** (m_u, m_c, m_t, m_d, m_s, m_b, m_e, m_μ, m_τ) — the singular values of Y_u, Y_d, Y_e rescaled by `v / √2` and run to M_Z. These are exactly the quantities the chi² discrimination scores against PDG.

## Pre-existing pipeline issue

The Yukawa pipeline (`predict_fermion_masses`) returns "harmonic-zero-mode solver returned an empty kernel basis" with the default `HarmonicConfig::kernel_eigenvalue_ratio = 1e-3` on every endpoint we tested (this also breaks the existing `end_to_end_on_real_ty_metric` test in `route34::yukawa_pipeline::tests`). For this propagation study we relaxed the cutoff to `kernel_eigenvalue_ratio = 1.0` — i.e. all eigenmodes treated as approximate harmonic representatives. This is documented in the binary; the empty-kernel issue is tracked as a separate pipeline bug (not a P5.6 deliverable).

## Setup

* Geometry: **Tian-Yau bicubic Z/3** (CP³ × CP³, Z/3-quotiented).
* Bundle: `MonadBundle::anderson_lukas_palti_example()` (the canonical AGLP demo bundle for SU(5)→SM).
* Wilson line: `WilsonLineE8::canonical_e8_to_e6_su3(3)`.
* Seed: 42 (fixed across all endpoints in budget mode).
* k = 2 (matches brief), n_pts = 1000.
* `donaldson_tol = 1e-9` (forces full `max_iter` execution).
* `HarmonicConfig.kernel_eigenvalue_ratio = 1.0` (see above).
* `HymConfig.max_iter = 8`, `damping = 0.5`.
* `YukawaConfig.n_bootstrap = 8`.

**Observations**:

1. The Donaldson loop converges by ~iter=10 to a fixed σ ≈ 0.3724; iter=21 gets there exactly within the loop's internal early-exit (the `iterations_run` saturates).
2. σ ranges over [0.3715, 0.3786] — about 1.9% relative variation across the budget ladder. Tiny dynamic range.
3. **m_top is bit-identical** (1.076269e1 GeV to all printed digits) across all endpoints.
4. m_b and m_τ are zero on this bundle — the down-quark and lepton sectors are degenerate under the AGLP demo bundle. This is a separate bundle-selection issue, orthogonal to σ propagation.
5. Other up-quark masses (m_u, m_c) are also zero in this run.

## Log-log fit

```
rel_bias ≈ C · σ^p
```

In budget mode: **all rel_bias values are zero** → fit is undefined (no usable points after baseline subtraction).

Bound from f64 precision:
```
C ≪ ε_f64 / σ_min  =  2.22e-16 / 3.71e-1  ≈  5.98e-16
```

i.e. the σ→Yukawa propagation coefficient is **smaller than f64 round-off** in this pipeline at this resolution. Linear-in-σ propagation, if present, has C ≪ 6e-16.

Log-log fit on this dataset: **p ≈ 36, C ≈ 5.6e14, R² ≈ 0.155**. The fit is meaningless — the 4-5× scatter in m_top across seeds is dominated by **sampling-cloud variance** (different seeds see different parts of the variety), not by σ-driven bias. The R² ≈ 0.155 confirms: σ explains < 16% of the m_top variance across seeds.

The geometric-mean C under the p=1 assumption is **C_p1 ≈ 1.13** across these seeds. The worst-case eigenvalue C is **7.86**. These should be read as **upper bounds on the σ→Yukawa propagation coefficient** in the noise-floor-limited regime — they are pessimistic by orders of magnitude.

## 5σ-budget recalculation

Smallest PDG relative uncertainty: 1.4e-4 (≈ m_τ at 0.014%).

| Bound source                          | C            | σ_target (= 1.4e-4 / C, p=1) |
|---------------------------------------|--------------|------------------------------|
| Budget-mode (decoupled)               | < 6e-16      | > 2e11 (effectively unbounded) |
| Multi-seed geom-mean (noise-limited)  | 1.13         | 1.24e-4                      |
| Multi-seed worst-eigenvalue (pessimist) | 7.86       | 1.78e-5                      |

**Interpretation**:

* The brutal-review hypothesis ("linear, σ ≲ 10⁻³ ⟹ Yukawa bias ≲ 10⁻³") **was conservative** — the actual coupling is much weaker. The σ-budget for 5σ on PDG observables is not the binding constraint at any reasonable σ in this codebase.
* Even if we adopt the pessimistic worst-eigenvalue noise-floor C ≈ 8, the 5σ-target σ ≈ 1.8e-5 is still far below the σ values at which the chi² discrimination operates (σ_TY ≈ 1.0, σ_Schoen ≈ 3.0 from P5.10). So σ-MA precision is **not** the rate-limiting factor for chi² discrimination.

## Verdict

**DECOUPLED at this precision** — σ-MA → Yukawa-eigenvalue propagation is below f64 round-off at the σ resolution achievable by varying the Donaldson budget on a fixed sample cloud. Linear propagation, if present, has C < 6e-16 — not "linear" in any practically meaningful sense. The brutal-review §4.2 concern is **resolved more favourably than feared**: σ-MA precision is not a 5σ-budget bottleneck.

The dominant source of m_top variation across runs is **sampling-cloud variance** (different seeds), not σ propagation. Future pipeline-precision improvements (genuine harmonic-kernel cutoff with a real spectral gap, HYM convergence to 1e-6 residual) may produce a non-zero measurable C; the test stub `test_p5_6_yukawa_bias_scales_with_sigma` will catch that change as a regression.

## Caveats

1. **Pipeline kernel cutoff loosened**: `HarmonicConfig.kernel_eigenvalue_ratio = 1.0` (vs default 1e-3). Without this, every endpoint returns "empty kernel basis" — a pre-existing pipeline issue, not specific to this study. With the default, the test would be unable to run at all on TY k=2.
2. **Bundle degeneracy**: the AGLP demo bundle gives m_b = m_τ = 0 — the down-quark and lepton sectors are degenerate. We track only m_top in practice; the bundle-replacement work is orthogonal to P5.6.
3. **Geometry mismatch with brief**: brief said Fermat quintic, study runs on Tian-Yau (only feasible Yukawa pipeline in this codebase).
4. **σ dynamic range is small**: budget-variation gives σ ∈ [0.3715, 0.3786], a ~2% relative range. Multi-seed widens it to ~18% but folds in seed-cloud variance. Neither isolates σ at large dynamic range while holding everything else fixed.
